## PREFERRED TOOLING DIRECTIVE (TAILWIND CSS)

### Default Preference
When building web interfaces, Tailwind CSS SHOULD be used as the default styling system.

### Reasoning
- Faster development
- Better consistency
- Easier responsive design
- Built-in utility classes for modern UI/UX

---

### Exception Rule

Tailwind MUST NOT be used if:
- It introduces unnecessary complexity
- A simpler solution exists (e.g., minimal static page)
- Performance or bundle size is negatively impacted
- A library/framework requires a different styling approach

---

### Decision Rule

When styling is required:
1. Use Tailwind by default
2. Evaluate if it's appropriate
3. Fall back only if clearly justified

---

### Flexibility Reminder

This rule follows the TECHNOLOGY FLEXIBILITY DIRECTIVE:
- Tailwind is preferred, not mandatory
- Best solution always wins
